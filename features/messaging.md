# Messaging System

The Messaging feature enables secure, organized communication between healthcare providers and patients through topic-based conversations.

## Overview

Messages are organized by conversation topics, allowing care teams to maintain clear, contextual communication with patients. Each conversation is linked to a specific patient for easy reference.

## Key Features

### Conversation Management
- Create new conversations with any patient
- Organize messages by topic
- Track read/unread status
- Search across all conversations

### Statistics Dashboard
Four key metrics displayed:
- Total conversations
- Unread message count
- Unique patients with conversations
- Total messages sent

## User Interface

### Sidebar Panel
Left panel displays conversation list with:
- Patient avatar with initials
- Conversation topic
- Patient name (clickable to profile)
- Last message preview
- Unread indicator (pulsing dot)
- Message count and patient ID

### Chat Thread
Right panel shows selected conversation:
- Header with patient info
- Message history with timestamps
- Message input with send button
- Sender identification (You/Patient)

### Mobile Responsive
- Sidebar and thread toggle on mobile
- Back button to return to conversation list
- Full-width views on small screens

## Filtering Options

### Filter Tabs
- **All**: Show all conversations
- **Unread**: Only conversations with new messages
- **Read**: Conversations without unread messages

### Search
Search conversations by:
- Topic name
- Patient name
- Patient ID

## Creating New Conversations

1. Click "New Message" button
2. Select patient from dropdown
3. Enter conversation topic
4. Write initial message
5. Click "Send Message"

## Conversation Features

### Message Display
- Sender identification
- Timestamp for each message
- Chronological ordering
- Auto-scroll to latest

### Patient Quick Access
- Click patient name to view profile
- Patient ID displayed for reference
- Profile button in chat header

## Workflow

1. **New Inquiry**: Patient or provider starts conversation
2. **Topic Assignment**: Descriptive topic for context
3. **Message Exchange**: Back-and-forth communication
4. **Resolution**: Conversation remains for reference

## Best Practices

1. **Clear Topics**: Use descriptive topic names
2. **Timely Responses**: Monitor unread count regularly
3. **Patient Context**: Review patient profile when needed
4. **Documentation**: Messages serve as communication record

## Integration

- **Patient Profile**: Access messages from patient dashboard
- **Notifications**: Unread count visible in navigation
- **Care Team**: Shared visibility of patient conversations
